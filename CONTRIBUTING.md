# ACMu への協力

ACMu は現在、小規模なプロジェクトです。
協力は歓迎しますが、変更範囲を小さく保ち、リポジトリのルールに従ってください。

## 作業前に読むもの

まず次を読んでください。

- `CLAUDE.md`
- `ARCHITECTURE.md`
- 作業対象の Issue

`CLAUDE.md` は現行の主要な AI エージェント運用ルールです。
別の AI エージェントを使う場合は、そのエージェント向けの常駐ルール文書も同じ内容に揃えてください。

実装に触れる場合は、関連する `src/acmu/ACMu.Core/` と `src/acmu/ACMu.PluginApi/` の契約も読んでください。

## プルリクエスト

外部協力者はプルリクエストを使ってください。

プルリクエストは小さく保ちます。

- 1 つのバグ、QA 修正、または機能の縦切りごとに 1 PR
- 挙動変更と大きなリファクタを混ぜない
- 生成 DLL/PDB を含めない
- ローカルに置くべきファイルを含めない
- 調査資料や設計資料は原則として含めない

## 停止条件

次に当たったら、作業を止めて確認してください。

- `src/acmu/ACMu.Core/` または `src/acmu/ACMu.PluginApi/` の変更が必要に見える
- Adapter 以外のモジュールから Besiege 本体/デコンパイル由来クラスを参照する必要がある
- 新しいアーキテクチャ判断が必要
- ゲーム外で確認できず、実機確認手順も定義されていない

## ビルド確認

通常は Visual Studio を使ってビルドします。`src/ACMu.sln` から実行してください。
もしくは次を使います。

```bash
./build.sh
```

協力者は原則として、Issue または PR に必要な確認項目を明記してください。
